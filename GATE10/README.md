# GATE10: Projection matrix generation for SPECT

This folder contains a Google Colab / Jupyter notebook workflow for **projection matrix generation for SPECT using GATE 10**, developed in the context of the **PERSPECT** project.

The notebook provides a practical and reproducible demonstration of how to:

- load an attenuation map from a DICOM file,
- construct voxelized attenuation and source maps for GATE,
- simulate dual-head SPECT projections with **OpenGATE / GATE10**,
- assemble a GATE-based projection matrix,
- and test reconstruction in the final section of the notebook.

The example included here uses data from an **anthropomorphic thyroid phantom**, making the workflow directly relevant to realistic SPECT system modelling and reconstruction studies.

---

## Contents of this folder

- **`PERSPECT_GATE10.ipynb`**  
  Main notebook containing the full demonstration workflow.

- **`GATE10.py`**  
  Python script used to run the GATE10 simulations for the selected projection angles.

- **`ATTMAPTOMOCT_EM001_DS.dcm`**  
  DICOM file containing the attenuation map of the anthropomorphic thyroid phantom.

- **`TOMOCT_EM001_DS.dcm`**  
  DICOM file containing the corresponding gamma-camera acquisition.

Additional files generated by the notebook may include:

- **`mu_map.mhd/.raw`**: voxelized attenuation map for GATE  
- **`source_map.mhd/.raw`**: voxelized source distribution for GATE  
- **`output_<angle>/`**: per-angle GATE simulation outputs, including projection files and statistics

---

## Quick start

To run the notebook:

1. Open `PERSPECT_GATE10.ipynb` in Google Colab or Jupyter.
2. Define the project folder through `PROJECT_PATH`.
3. Make sure the required files are available in that directory.
4. Run the setup cells to install dependencies and import the required modules.
5. Load the attenuation-map DICOM file and generate the `.mhd/.raw` files.
6. Run the GATE simulation over the selected projection angles.
7. Assemble the projection matrix and test reconstruction in the final section.

## Notes

### Dual-head acquisition
Only projection angles smaller than **180°** are simulated in the demonstration because the setup is **dual-head**. For each camera position, the two detector heads are placed **180° apart**, so the range **0°–180°** already provides the full angular coverage.

### Material segmentation
The attenuation map is segmented into a small number of material classes for demonstration purposes. Users may define a more detailed material segmentation, but in that case the corresponding `phantom.voxel_materials` mapping in `GATE10.py` must also be updated so that the voxel labels remain consistent with the materials assigned in GATE.

### Demonstration activity and runtime
For demonstration purposes, a **low source activity** is used in the GATE simulation in order to keep runtimes manageable in Colab. Practical applications may require higher activity and more efficient batch execution.

### Reconstruction
GPU acceleration is strongly recommended for the reconstruction part of the notebook. In practice, the free GPU resources available in Google Colab are sufficient for demonstration and testing.
